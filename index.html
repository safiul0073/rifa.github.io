Getting Started

Welcome to Rifa | A Lottery Platform We would like to thank you for choosing our Product - Rifa .

It's built with Reactjs + Nextjs as frontend and Laravel + Vuejs as backend It's fully responsive and well tested also fully functional. It can be easily customized. We create API'S for frontend and Admin panel we used Inertiajs.

Requirements

We recommend you to install this project on VPS or Rifa Admin on cpanel and frontend on varcel. But you can do admin and frontend it on cpanel or any managed server (We don't recommend because of quite hard to upload). Our project divided into two parts - Frontend and Backend. So for this project you need to have the following requirements on your system.

  • PHP 8.3 ^ (recommended 8.3)
  • MySQL 8.0 ^ or MariaDB 10.6 ^
  • NodeJS 20 ^ (recommended 22)
  • Composer
  • Intl PHP Extension
  • Curl PHP Extension
  • GD PHP Extension
  • Gettext PHP Extension
  • GMP PHP Extension
  • Mbstring PHP Extension
  • Exif PHP Extension
  • MySQL PHP Extension
  • Pdo_mysql PHP Extension
  • Zip PHP Extension
VPs requirements
  • RAM: 2GB
  • Disk: 4GB
  • CPU: 1
Cpanel requirements
  • RAM: 4GB
  • Disk: 4GB
  • Cloud Linux (it's recommended when you want to upload admin and frontend, But it's not recommended if just upload admin panel)

Using XAMPP for PHP, MySQL, and Apache on Windows

  1. Download XAMPP for Windows.
  2. Run the installer and follow the prompts, ensuring Apache, MySQL, and PHP are selected.
  3. Open the XAMPP Control Panel and start Apache and MySQL .
  4. Create a PHP test file:
    1. Go to C:\xampp\htdocs and create a file named info.php with the following content: 
    <?php phpinfo(); ?>
    1. Open http://localhost/info.php in a browser to confirm PHP is installed.
  5. Access MySQL through phpMyAdmin , available at http://localhost/phpmyadmin .

Installing Composer

  1. Download Composer-Setup.exe .
  2. Run the installer and ensure the XAMPP PHP path ( C:\xampp\php ) is in your system's PATH.
  3. Verify Composer installation by running: 
    composer -v

Installing Node.js

  1. Go to the Node.js download page and download the Windows installer (LTS version recommended).
  2. Run the installer and follow the setup instructions. Make sure to check the option to add Node.js to PATH.
  3. Verify installation by running the following commands in the Command Prompt: 
    node -v
    npm -v

Installing our Rifa Admin

  1. Open the XAMPP Control Panel and start Apache and MySQL
  2. Access MySQL through phpMyAdmin, available at http://localhost/phpmyadmin
  3. Create a new database:
  4. Go to our Rifa directory then copy the admin folder to C:\xampp\htdocs folder.
  5. Go to C:\xampp\htdocs\admin
  6. open a terminal and run composer install
  7. Run the following commands in the terminal: 
    composer install
  8. install node modules with npm 
    npm install --force
  9. Run the following commands in the terminal to start the development server: 
    npm run dev
  10. Then open another terminal and run the following command to start the development server: 
    php artisan serve
    The project will be accessible at http://127.0.0.1:8000 .

Now you can see an installer page with 4 step for the Rifa Admin.

  1. Terms of use for this project
  2. Server requirements, Ensure the server requirements are correct
  3. Folder permissions, Ensure the folder permissions are correct
  4. Installations and configurations, At first add application URL and Application Name and then Database Configuration also add an admin user.

Installing our Next.js Frontend Project

  1. Go to our Rifa directory then frontend folder
  2. open a terminal and install dependencies 
    npm install --force
  3. Copy .env.example to .env.local and update the database details 
    cp .env.example .env.local
  4. Edit the .env.local file and add
    • NEXT_PUBLIC_SERVER_URL="http://127.0.0.1:8000" (Admin URL)
    • NEXT_PUBLIC_BASE_URL="http://localhost:3000" (Frontend URL)
    • NEXT_PUBLIC_TOKEN_NAME="test"
  5. Run the following command in the terminal to start the development server: 
    npm run dev
    The project will be accessible at http://localhost:3000 .

Step 9: Setting Up Server & Project

In this section, we'll set up our server and also will set up Reverse Proxy to host all of our sites from the same server.

At first, we'll disable the default configuration.

sudo rm /etc/nginx/sites-enabled/default
Step 10: Create New Nginx for the domain

                                            sudo touch /etc/nginx/sites-available/rifa
                                            sudo nano /etc/nginx/sites-available/rifa

Add this Nginx config file to that edited file,


                                            server {
                                                listen 80;
                                                server_name YOUR_DOMAIN.com;
                                                add_header X-Frame-Options "SAMEORIGIN";
                                                add_header X-XSS-Protection "1; mode=block";
                                                add_header X-Content-Type-Options "nosniff";
                                                index index.html index.htm index.php;
                                                charset utf-8;
                                                # For admin
                                                location /admin {
                                                alias /var/www/rifa/admin/public;
                                                try_files $uri $uri/ @admin;
                                                location ~ \.php$ {include fastcgi_params;
                                                fastcgi_param SCRIPT_FILENAME $request_filename;
                                                fastcgi_pass
                                                unix:/run/php/php8.3-fpm.sock;
                                                }
                                                }
                                                location @admin {
                                                rewrite /admin/(.*)$ /admin/index.php?/$1 last;
                                                }
                                                # For FrontEnd -> Rest
                                                location /{
                                                proxy_pass http://localhost:3000;
                                                proxy_http_version 1.1;
                                                proxy_set_header Upgrade $http_upgrade;
                                                proxy_set_header Connection 'upgrade';
                                                proxy_set_header Host $host;
                                                proxy_cache_bypass $http_upgrade;
                                                }
                                                
                                                error_page 404 /index.php;
                                                location ~ \.php$ {
                                                fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
                                                fastcgi_index index.php;
                                                fastcgi_param SCRIPT_FILENAME
                                                $realpath_root$fastcgi_script_name;
                                                include fastcgi_params;
                                                }
                                                location ~ /\.(?!well-known).* {
                                                deny all;
                                                }
                                            }
Make sure you change YOUR_DOMAIN.com to your specific domain name You can change admin path, if you want to change the the domain path for the laravel application

Save and close the file by typing CTRL and X, then Y and ENTER when you are finished.

Then enable the config

sudo ln -s /etc/nginx/sites-available/rifa /etc/nginx/sites-enabled/

Make sure you didnt introduce any syntax errors by typing: 

sudo nginx -t

Next, restart Nginx:

sudo systemctl restart nginx

Cpanel Deployment

It's quite hard to debug any deployment issue on the cPanel or any managed server as the provider manages this type of server, and they've complete control of the server. Cache Server

Cache is a mechanism used to store frequently accessed data temporarily to reduce the time and resources required to retrieve it. Caching significantly improves the performance of an application by minimizing database queries or API calls for the same data.

In Laravel, you can use Redis (recommended) or Memcached as cache servers. Both are free and open-source caching solutions.

Important: Redis or Memcached can only be installed on a VPS or similar hosting environment where you have root access.

Redis Installation

For Ubuntu

  1. Update the System: 
                              
                            sudo apt update
                            sudo apt upgrade
  2. Install Redis: 
    sudo apt install redis-server
  3. Enable Redis to Start on Boot: 
    sudo systemctl enable redis
  4. Start Redis Service: 
    sudo systemctl start redis
  5. Verify Installation: 
    redis-cli ping

    The response should be:

    PONG
  6. Secure Redis (Optional): 
    sudo nano /etc/redis/redis.conf

    Uncomment and set a strong password:

    requirepass YourStrongPassword

    Save and restart Redis:

    sudo systemctl restart redis

Laravel Configuration for Redis

  1. Update the .env file: 
    
                          CACHE_STORE=redis
                          REDIS_CLIENT=predis
                          REDIS_HOST=127.0.0.1
                          REDIS_PASSWORD=YourStrongPassword
                          REDIS_PORT=6379
  2. Install the PHP Redis extension: 
    sudo apt install php-redis
  3. Restart the web server: 
    
                  sudo systemctl restart apache2   # For Apache
                  sudo systemctl restart nginx    # For Nginx

Memcached Installation

For Ubuntu

  1. Update the System: 
    sudo apt update
  2. Install Memcached: 
    sudo apt install memcached libmemcached-tools
  3. Start Memcached: 
    sudo systemctl start memcached
  4. Enable Memcached to Start on Boot: 
    sudo systemctl enable memcached
  5. Verify Memcached is Running: 
    echo stats | nc 127.0.0.1 11211

Laravel Configuration for Memcached

  1. Update the .env file: 
    
                            CACHE_DRIVER=memcached
                            MEMCACHED_HOST=127.0.0.1
                            MEMCACHED_PORT=11211
  2. Install the PHP Memcached extension: 
    sudo apt install php-memcached
  3. Restart the web server: 
    
                  sudo systemctl restart apache2   # For Apache
                  sudo systemctl restart nginx    # For Nginx

After Installation Clear Application Cache

  1. Clear the Cache: 
    php artisan cache:clear

Choosing Between Redis and Memcached

Redis is preferred for complex use cases, persistence, and when advanced features are needed.
Memcached is suitable for simpler, lightweight caching needs where speed is the primary concern.

Laravel Octane Setup Guide

Laravel Octane supercharges your application by serving requests via high-performance application servers such as Swoole or RoadRunner. It significantly improves performance by keeping the application in memory across multiple requests.

Prerequisites

  • PHP 8.3 or higher. (recommended 8.3)
  • Composer installed on your system.
  • A Laravel application set up.

Installation

  1. Install Laravel Octane:

    Our project already have it installed.

  2. Publish Octane Configuration:

    Already published octane configuration. you can check it on config/octane.php

    This will install Octane and allow you to select a server (e.g., Swoole or RoadRunner). Default Selected on config is roadrunner. You can change it on .env file OCTANE_SERVER

  3. Install Your Selected Server:

    For Swoole:

    composer require swoole/ide-helper

    For RoadRunner:

    composer require spiral/roadrunner

Configuration

  1. Update the Environment File:

    Add the following configuration to your .env file:

    
                          OCTANE_SERVER=swoole   # or roadrunner
                          OCTANE_HTTPS=false
                          OCTANE_MAX_REQUESTS=500
                          OCTANE_WORKERS=auto
  2. Update the Octane Configuration File:

    Edit the config/octane.php file if you need to customize the default settings.

Starting the Octane Server

  1. Run the Server: 
    php artisan octane:start

    2. Importent: If you getting error on octane:start such as extension not found. you need to install Swoole or Roadrunner

    For Swoole: 

    
                        # Install Swoole as PHP extension
                        sudo apt install php8.3-swoole
                        # Verify Swoole is loaded
                        php -m | grep swoole

    For RoadRunner: 

    
                          # Install RoadRunner as PHP extension
                          wget https://github.com/roadrunner-server/roadrunner/releases/latest/download/roadrunner-linux-amd64 -O roadrunner
                          # make it executable
                          chmod +x roadrunner
                          sudo mv roadrunner /usr/local/bin/roadrunner
                          # Verify RoadRunner is loaded
                          roadrunner --version

  2. Run with HTTPS (Optional): 
    php artisan octane:start --https
  3. Run in Production Mode: 
    php artisan octane:start --env=production

Monitoring and Debugging

  • Use Laravel Telescope for debugging requests.
  • Monitor server logs for performance insights.

You can access it on YOUR_ADMIN_URL/telescope Example: http://localhost/telescope

Additional Notes

  • Ensure Swoole or RoadRunner dependencies are installed on your server.
  • For production environments, configure a process manager like supervisor to keep the Octane server running.
  • Octane is optimized for long-running processes. Avoid using facades that bind to the service container for each request (e.g., Auth, Session).

Stopping the Octane Server

To stop the server, use:

php artisan octane:stop

CI/CD with GitHub Actions and Laravel Envoy

This guide explains how to set up Continuous Integration and Continuous Deployment (CI/CD) for a Laravel application using GitHub Actions and Laravel Envoy. This ensures smooth automated deployments to your server whenever new code is pushed to the repository.

Prerequisites

  • A Laravel project hosted on a GitHub repository.
  • Laravel Envoy installed globally: 
    composer global require laravel/envoy
  • A server (e.g., VPS or dedicated server) accessible via SSH. (Check this guide)
  • Set up SSH keys for passwordless access between the CI/CD pipeline and the server. Follow this guide

Step 1: Laravel Envoy Setup

Envoy allows you to define deployment tasks in a clean, simple syntax.

  1. Ensure you have an Envoy.blade.php file in the Laravel project root. Update it with the following content:
    2. <?php
                  
                  @servers(['prod' => ['your_username@your_server']])
                  
                  @setup
                      $REPO_URL = 'git@github.com:username/repository.git'; // Update repository URL
                      $WORK_DIR = '/var/www/rifa/admin'; // Update project directory
                      $REPO_BRANCH = 'main'; // Update branch
                      $BACKUP_DIR = '/var/www/rifa/backups';
                      $TIMESTAMP = date('Y-m-d-His');
                  
                      function logMessage($message) {
                          return "echo '\033[32m" .$message. "\033[0m';\n";
                      }
                  
                      function logError($message) {
                          return "echo '\033[31m" .$message. "\033[0m';\n";
                      }
                  @endsetup
                  
                  @story('deploy')
                      start-agent
                      backup
                      update-code
                      install-dependencies
                      set-permissions
                      migrate-db
                      restart-workers
                  @endstory
                  
                  @story('rollback')
                      start-agent
                      restore-backup
                      restore-db
                      set-permissions
                      restart-workers
                  @endstory
                  
                  @task('start-agent')
                      {{ logMessage("Starting SSH agent") }}
                      eval "$(ssh-agent -s)"
                      ssh-add ~/.ssh/id_github
                  @endtask
                  
                  @task('backup')
                      {{ logMessage("Creating backup") }}
                      mkdir -p {{ $BACKUP_DIR }}/{{ $TIMESTAMP }}
                      cd {{ $WORK_DIR }}
                      git rev-parse HEAD > {{ $BACKUP_DIR }}/{{ $TIMESTAMP }}/git_commit
                      php artisan backup:run --only-db --backup-name={{ $TIMESTAMP }}
                      cp composer.lock {{ $BACKUP_DIR }}/{{ $TIMESTAMP }}/composer.lock
                      cp package.json {{ $BACKUP_DIR }}/{{ $TIMESTAMP }}/package.json
                  @endtask
                  
                  @task('update-code')
                      {{ logMessage("Updating code") }}
                      cd {{ $WORK_DIR }}
                      git stash
                      git pull origin {{ $REPO_BRANCH }}
                  @endtask
                  
                  @task('install-dependencies')
                      {{ logMessage("Installing dependencies") }}
                      cd {{ $WORK_DIR }}
                      composer install --no-interaction --optimize-autoloader
                      php artisan optimize
                  @endtask
                  
                  @task('set-permissions')
                      {{ logMessage("Setting permissions") }}
                      sudo chown -R "${USER}:www-data" {{ $WORK_DIR }}
                      sudo chmod -R 775 {{ $WORK_DIR }}
                  @endtask
                  
                  @task('migrate-db')
                      {{ logMessage("Running database migrations") }}
                      php {{ $WORK_DIR }}/artisan migrate --force
                  @endtask
                  
                  @task('restart-workers')
                      {{ logMessage("Restarting workers") }}
                      sudo supervisorctl restart all
                      php {{ $WORK_DIR }}/artisan queue:restart
                  @endtask
                  
                  @task('restore-backup')
                      {{ logMessage("Restoring from backup") }}
                      $RESTORE_TIME = isset($TIMESTAMP) ? $TIMESTAMP : trim(shell_exec("ls -t {{ $BACKUP_DIR }} | head -n1"));
                      cd {{ $WORK_DIR }}
                      if [ -f "{{ $BACKUP_DIR }}/$RESTORE_TIME/git_commit" ]; then
                          {{ logMessage("Restoring code to previous commit") }}
                          git fetch --all
                          git reset --hard $(cat {{ $BACKUP_DIR }}/$RESTORE_TIME/git_commit)
                      else
                          {{ logError("No git commit backup found") }}
                      fi
                      if [ -f "{{ $BACKUP_DIR }}/$RESTORE_TIME/composer.lock" ]; then
                          {{ logMessage("Restoring composer dependencies") }}
                          cp {{ $BACKUP_DIR }}/$RESTORE_TIME/composer.lock ./
                          composer install --no-interaction --optimize-autoloader
                      fi
                  @endtask
                  
                  @task('restore-db')
                      {{ logMessage("Restoring database") }}
                      $RESTORE_TIME = isset($TIMESTAMP) ? $TIMESTAMP : trim(shell_exec("ls -t {{ $BACKUP_DIR }} | head -n1"));
                      php artisan backup:restore --backup-name=$RESTORE_TIME
                  @endtask
                  
                  @finished
                      {{ logMessage("Deployment complete") }}
                  @endfinished
  2. Test the deployment manually with: 
    envoy run deploy

Step 2: GitHub Actions Setup

GitHub Actions for Laravel Deployment

GitHub Actions automates the process of deploying your Laravel application. By configuring workflows, you can trigger deployments whenever code is pushed to your repository.

Prerequisites
  • GitHub repository containing your Laravel project.
  • Laravel Envoy setup for deployment. Refer to the CI/CD with Laravel Envoy guide.
  • SSH access configured between GitHub Actions and your server (passwordless SSH setup).
  • Private SSH key added as a GitHub secret.
Step 1: Create Workflow File

GitHub Actions workflows are defined in YAML files stored in the .github/workflows directory of your repository.

  1. Create the workflows directory: 
    mkdir -p .github/workflows
  2. Create the deployment workflow file: 
    touch .github/workflows/deploy.yml
  3. Edit the deploy.yml file with the following content:
    4. 
                          deploy:
                          runs-on: ubuntu-latest
                      #    needs: [ test, build ]
                          steps:
                            - name: Checkout code
                              uses: actions/checkout@v2
                            - name: Install SSH Key
                              uses: shimataro/ssh-key-action@v2
                              with:
                                key: ${{ secrets.SSH_PRIVATE_KEY }} # SSH private key content from GitHub secrets
                                known_hosts: 'just-a-placeholder-so-we-dont-get-errors'
                            - name: Adding Known Hosts
                              run: ssh-keyscan -H YOUR_SERVER_IP >> ~/.ssh/known_hosts #example: ssh-keyscan -H 237.84.2.178 >> ~/.ssh/known_hosts
                            - name: Setup PHP
                              uses: shivammathur/setup-php@v2
                              with:
                                php-version: 8.3
                                tools: composer:v2
                                coverage: none
                            - name: Cache composer dependencies
                              uses: actions/cache@v3
                              with:
                                path: vendor
                                key: composer-${{ hashFiles('composer.lock') }}
                            - name: Install Project Dependencies
                              run: composer install --no-interaction --prefer-dist --optimize-autoloader
                            - name: Deploy Environment
                              run: ./vendor/bin/envoy run deploy
Step 2: Configure GitHub Secrets

Add the necessary secrets to your GitHub repository:

  1. Navigate to your repository on GitHub.
  2. Go to Settings > Secrets and Variables > Actions.
  3. Add a new secret:
    • Name: SSH_PRIVATE_KEY
    • Value: Your private SSH key.
Step 3: Trigger Deployment
  1. Push code to the main branch (or the branch you configured): 
                                
                              git add .
                              git commit -m "Trigger deployment"
                              git push origin main
  2. Monitor the workflow execution under the Actions tab in your repository.
  3. Ensure all steps complete successfully and verify the deployment.
Additional Notes
  • Always test workflows in a staging environment before deploying to production.
  • Ensure your server has the necessary permissions and tools installed (e.g., Composer, PHP, SSH).
  • Monitor the Actions tab for workflow logs and troubleshoot issues as needed.
Step 3: Testing the CI/CD Pipeline
  1. Push changes to the repository and monitor the pipeline under the Actions tab.
  2. Test rollback functionality using: 
    envoy run rollback

Additional Notes

  • Ensure the correct paths and branch names are configured.
  • Verify SSH key configuration and server permissions.
  • Keep backup files secure and clean up old backups periodically.

System Configuration

In this section you can change some configuration enabled or disabled also frontend pagination settings

  1. From The Admin Panel Go To Settings System Configurations

You can change following configurations:

  • User Registration switch is enabled then any your can create new account
  • Agreement & Trams Policy switch is enabled then user need to accept terms and conditions for account creation
  • Force Secure Password switch is enabled then user need to enter secure password for account creation other wise user can create account without secure password
  • Email Verification switch is enabled then user need to verify email address
  • Phone Verification switch is enabled then user need to verify phone number
  • Email Notification switch is enabled then user will get notification on email for every new order, welcome, bonus etc
  • Phone Notification switch is enabled then user will get notification on phone for every new order, welcome, bonus etc

Notification Setting

In this section have Four setting, one for Email Service, one for SMS Service, one for Popup Realtime Notification and Notification Template.

  1. From The Admin Panel Go To Settings Notification Setting

Email Smtp Settings, you can add or update smtp Credentials by following this screenshot, Go your mail server provider and get the credentials and past it here.

Sms Twilio Settings, you can add or update Twilio Credentials by following this screenshot. Go your Twilio account and get the credentials and past it here. It's international service.

Real time notification settings. You need to a reverb (Websocket) server and you will only use on VPS server. You just need to create a reverb server then get you credentials and past it here. Follow This Tutorial

  • Key is just Random Generated String
  • Secret is just Random Generated String
  • Also App Id is a random generated string
  • Host is your reverb server url
  • Port is your reverb server port
  • Channel is your reverb server channel such as http or https
